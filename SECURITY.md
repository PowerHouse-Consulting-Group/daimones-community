# Security Policy

## Supported Versions

We take the security of daïmōnes and its community very seriously. Currently, we apply security patches and updates to the active production branch.

| Version | Supported          |
| ------- | ------------------ |
| v1.x.x  | :white_check_mark: |
| Pre-v1  | :x:                |

## Reporting a Vulnerability

**DO NOT report security vulnerabilities through public GitHub issues.**

If you discover a potential security vulnerability in daïmōnes, please report it immediately by emailing us at:
**architect@daimones.ai**

Please include the following information in your report:
- Type of vulnerability (e.g., XSS, SQLi, Prompt Injection).
- Steps to reproduce the issue.
- Potential impact of the vulnerability.

We will acknowledge receipt of your vulnerability report within 48 hours and strive to send you regular updates about our progress. Once the issue is resolved, we will publish a security advisory.

## AI & Data Privacy
Given that daïmōnes operates as an institutional dialectic engine processing scholarly data:
- We treat all data sovereignty and privacy bypass issues as Critical security vulnerabilities.
- For prompt injection or jailbreak attempts against the Aristotle persona, please submit these through our standard feedback channels unless they expose underlying infrastructure.
